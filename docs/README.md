Found something missing or incorrect?

1. **Small fixes:** Edit the `.md` file directly and submit PR
2. **New sections:** Follow existing structure and style
3. **Questions:** Create GitHub issue

### Documentation Style Guide

- Use **clear headings** (H1 for title, H2 for major sections)
- Include **code examples** with syntax highlighting
- Add **tables** for structured information
- Use **emoji sparingly** (only for status indicators)
- Link to **related documentation**
- Keep **paragraphs short** (3-5 sentences max)
